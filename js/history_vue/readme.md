JS 历史
 - const 
 常量， es5 没有专门的常量申明
 简单数据类型 值不可以改变
 对象呢？值可以改变，它指向的内存地址不可以改变。
 （const 声明的变量不能被重新赋值，但如果变量是对象或数组，其属性或元素可以被修改。）

 - 赋值 
 简单数据类型 值传递
 复杂数据类型 地址传递（引用式赋值）
const 类型不同处理方式不同

内存栈 连续， 空间小 快
内存堆 不连续 空间比较大
const 简单数据，复杂数据虽然表现不一致，内存栈中的值不会发生改变，但是堆中的值会发生改变。

es6 第一个特性 let const，弥补了es5只有var的不足，支持块级作用域，支持常量，
